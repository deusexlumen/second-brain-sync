# Marr — Vision: Eine Computational Investigation (1982)

## Dekonstruierter Kern-Gedanke
Marr zerlegt kognitive Systeme in drei analytische Ebenen: **Computational** (Was berechnet das System? Welches Problem wird gelöst?) → **Representation & Algorithm** (Wie wird es repräsentiert? Welcher Algorithmus transformiert Input zu Output?) → **Hardware Implementation** (Womit wird es physikalisch realisiert?). Diese Trennung ist revolutionär, weil sie **Multiple Realisierbarkeit** ermöglicht: Dieselbe Computation kann in Neuronen, Transistoren oder Zahnrädern laufen. Marr's Vision-Analyse (Primal Sketch → 2.5D Sketch → 3D Model) beschreibt eine Architektur, die Deep Learning erst 30 Jahre später implementierte — ein Beweis für die zeitlose Relevanz der Drei-Ebenen-Analyse.

## Ariadne-Brücken (Cross-Domain Links)
- **Isomorphe Struktur:** [[hoppe-zeitpraeferenz-entzivilisierung]] -> *Mechanik:* Marr's Drei-Ebenen-Analyse trennt **Funktion** (was) von **Mechanismus** (wie) von **Substrat** (womit). Hoppe's Analyse trennt **Ziel** (Zivilisation) von **Mechanismus** (Zeitpräferenz) von **Substrat** (Eigentumsregime). Beide sind **Multi-Level-Theorien**, die verhindern, dass Ebene-3-Änderungen (Hardware/Regime) als Ebene-1-Änderungen (Computation/Zivilisation) missverstanden werden. Ein Wechsel von Monarchie zu Demokratie (Hardware) ändert nicht das Ziel (Zivilisation), aber die Realisierung.
- **Spannungsfeld / Paradoxon:** [[turing-computing-machinery-1950]] -> *Konflikt:* Turing's Imitation Game operiert ausschließlich auf Marr's oberster Ebene (Computational/Output). Es testet, ob die **Funktion** äquivalent ist, ignoriert Repräsentation und Hardware. Marr würde argumentieren: Der Turing-Test ist unvollständig — er verfehlt die entscheidende Frage "Wie funktioniert es?" Aber: Wenn Multiple Realisierbarkeit gilt, warum sollte "Wie" überhaupt relevant sein? Die Auflösung: Der Turing-Test ist ein **Ebene-1-Test**; Marr fordert **Ebene-2-Erklärungen**. Beide sind notwendig, aber für verschiedene Zwecke (Validierung vs. Verstehen).

## Synthetischer Impuls (Emergenz)
> Wenn wir Marr's Drei-Ebenen-Analyse auf **magische Systeme** anwenden: Computational = Welches Bewusstseins-Problem wird gelöst? (z.B. Trance-Induktion); Representation/Algorithm = Welche Symbol-Manipulation erzeugt den Zustandswechsel? (z.B. Enochian Calls); Hardware = Welches Substrat realisiert die Operation? (z.B. menschliches Nervensystem + akustische Resonanz). Könnten wir dann "Multiple Realisierbarkeit" für Magie postulieren — dieselbe Trance-Computation in verschiedenen Substraten (Schamanentrommel, Enochian Calls, binaurale Beats)?
